SLAMAdverserialLab

An Extensible Framework for Visual SLAM Robustness Evaluation under Adverse Conditions

Mohamed Hefny1 Steven Ko1 Karthik Dantu2
1Simon Fraser University 2University at Buffalo
Paper Code BibTeX

Abstract

Visual SLAM systems now reach high accuracy on clean datasets, but we still lack an end-to-end, configurable way to stress-test complete pipelines under challenging visual and video-transport conditions. We present SLAMAdverserialLab, a modular framework that takes monocular or stereo RGB sequences from arbitrary datasets, applies user-specified, physically parameterized perturbations, runs one or more SLAM systems on the perturbed sequences, and provides pipelines for trajectory- and VO-level evaluation.

We instantiate three condition families: scene and illumination (rain in mm/h, fog via visibility, day-to-night shifts), optics (lens soiling, cracked lens occlusions, vignetting, blur), and video transport (bitrate-induced compression and packet-loss-like frame drops). By perturbing existing datasets instead of collecting new ones, the framework lets users compose customizable perturbation suites tailored to the specific conditions they want to test.

Contributions

  • An end-to-end framework for systematic robustness evaluation of Visual SLAM systems under adverse conditions, requiring no new data collection by perturbing existing benchmark datasets.
  • A modular perturbation system with 13 modules across three families: scene/illumination (fog, rain, day-night), optics (lens soiling, cracks, vignetting, blur), and video transport (compression, frame drops).
  • Physics-based parameterization using real-world units for reproducibility: fog visibility in meters via the Koschmieder model, rain intensity in mm/h, and video bitrate in kbps.
  • Multi-algorithm evaluation through a unified interface supporting ORB-SLAM3, DROID-SLAM, MASt3R-SLAM, DPVO, MonoGS, and related baselines.
  • Composable perturbation chains enabling sequential composition to study compound degradation scenarios that reflect real-world failure modes.
  • Standardized evaluation with trajectory metrics, tracking statistics, and automated comparison plots via the evo library.
